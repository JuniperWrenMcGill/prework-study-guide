CCS

  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • There are three ways to apply CSS:
    Inline CSS: Applied directly in HTML using the style attribute. Quick changes, but less flexible
    Internal CSS Style Sheet: Embedded in HTML using the <style> element. Rules apply to specific page elements.
    External CSS Style Sheet:Preferred method for most cases. CSS rules kept in a separate file for easier design changes. Linked to HTML using <link> in the <head>
  • Steps to link the external CSS file to the HTML using the <link≫ element.
    1. Open your HTML file in the text editor.
    2. Locate the <head> section of the HTML file.
    3. Insert the following <link> element after the final <meta> element and before the <title> element:
    <link> rel="stylesheet" href="./assets/style.css">
  • How to use .card class to syle each topic for better visibility.

    3. Save CSS file.
    4. Open index.html, ensure eachhas class="card".
    5. Add notes to the guide using <ul> and <li>.
    6. Save HTML file

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • Article on Markdown Syntax: Basic Writing and Formatting Syntax
  • How to Create a New Repository:
    1. Log in to GitHub.
    2. Click the plus sign near your profile picture.
    3. Choose "New Repository" from the dropdown.
    4. Name your repository (e.g., "prework-study-guide").
    5. Add a description and make it Public.
    6. Select "Add README" and choose a license.
    7. Review and ensure options are selected.
    8. Click "Create Repository" and explore your new repository.
  • GitHub Issues: Purpose: Manage and track tasks for the project.
  • How to Create New Issues:
    1. Navigate to the repository on GitHub.
    2. Click on the "Issues" tab.
    3. Click the "New Issue" button.
    4. Add a title.
    5. Use Markdown to create a detailed description.
    6. Submit the issue.
  • GitHub Flow:
    1. Create a Feature Branch: Start with a feature branch, not directly in the main branch.
    2. Make Changes and Commit: Make changes in the feature branch and commit them. Use descriptive commit messages.
    3. Push Changes to GitHub: Push changes from your local feature branch to GitHub.
    4. Open a Pull Request (PR): Open a PR to merge the feature branch into the main branch. PR compares your changes (feature branch) with the main branch.
    5. Code Review and Approval: Others review your changes, comment, and suggest. Resolve conflicts if any.
    6. Merge into Main: Once approved, merge your changes into the main branch. Main branch is now updated with your changes.
  • Syncing Local and Remote Branch:
    1. Use the command line to go to the project directory.
    2. Create Feature Branch: "git checkout -b feature/starter-code", "git add -A", "git commit -m "add starter code", "git pull origin main", "git push origin feature/starter-code",
    3. In the browser, go to the GitHub repository. Select the "Compare & pull request" button.
    4. Merge PR: Add a title, leave the body empty. Set base branch to "main", comparison branch to "feature/starter-code."" Click "Create Pull Request" and then "Merge Pull Request."
    5. Verify Changes: Check the main branch on GitHub to see your changes.

Javascript

  • Article on JavaScript Basics: Getting Started with JavaScript
  • Article on JavaScript Data Types and Data Structures: JavaScript data types and data structures
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Data Types: The classification of values in JavaScript, such as strings, numbers, booleans, etc.
  • String: A sequence of characters, typically used for text.
  • Number: Numeric data type, including integers and floating-point numbers.
  • Boolean:A data type representing true or false values.
  • Array: An ordered list of values, accessed by index.
  • Object:A collection of key-value pairs, representing a real-world entity.
  • Function: A reusable block of code, designed to perform a specific task.
  • Conditional Statement: A statement that executes different code based on specified conditions.
  • Loop: A programming construct that repeats a set of instructions until a specific condition is met.
  • Event: An action or occurrence recognized by a program, often triggering a response.
  • DOM (Document Object Model): A programming interface for web documents, allowing manipulation of HTML and XML documents.
  • Callback: A function passed as an argument to another function, to be executed later.
  • Closure: A function that "closes over" variables from its outer scope, retaining access to them.
  • Asynchronous: Code execution that doesn't block the program, often used for tasks like fetching data or handling events.
  • API (Application Programming Interface): A set of rules and tools for building software applications.
  • JSON (JavaScript Object Notation): A lightweight data interchange format, often used for data exchange between a server and a web application.
  • AJAX (Asynchronous JavaScript and XML): A technique for updating parts of a web page without reloading the entire page.
  • ES6/ES2015:The sixth edition of the ECMAScript standard, introducing new features to JavaScript.
  • Debugging: The process of finding and fixing errors in code
  • Some Integrating Basics:
    1. Integrating JavaScript into a Project:
    • Navigate to the project directory in the command line.
    • Ensure you are on the main branch using git checkout main and synchronize with the latest changes using git pull origin main.
    • Create a new feature branch with git checkout -b feature/add-js.
    • Go to the relevant folder (e.g., assets) and create a new file, for instance, script.js.
    • Open the newly created file in your preferred code editor.
    2. Creating a JavaScript Variable:
    • Declare a variable in the JavaScript file with a meaningful name and assign a value (e.g., var myVariable = "example").
    • Utilize console.log(myVariable) to display the variable's value in the console.
    • Save the file.
    3. Connecting JavaScript File to HTML:
    • Open the HTML file and include <script src="./path/to/your/script.js"></script> under the <footer> and above the closing </body> tag
    • Save the HTML file.
    4. Using Browser Developer Tools:
    • Open the HTML file in a browser using a development server.
    • Access the browser's Developer Tools by right-clicking and selecting "Inspect" or using keyboard shortcuts.
    • Navigate to the Console tab to inspect the output of console.log.
  • Overview of Control Flow:
    1. Control flow determines the order in which a computer executes code.
    2. JavaScript code runs sequentially, from the first statement at the top to the last at the bottom.
